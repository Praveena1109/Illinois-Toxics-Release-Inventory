## 1. Project Overview

The **Toxics Release Inventory (TRI)** is an annual self-reporting program under EPCRA Section 313, requiring industrial facilities above threshold levels to report releases of over 800 toxic chemicals to air, water, and land. This project curates the **Illinois subset** of TRI Basic Data Files for 2010–2024.

Raw TRI files are publicly available but not independently understandable. This project transforms them into a structured, documented, reproducible dataset suitable for environmental research, policy analysis, and educational use.

**Primary contributions:**
- Integration of 15 annual files into a single longitudinal dataset
- Systematic quality assessment
- Seven curation flags applied to every record
- Five derivative datasets for common analytical use cases
- Full data dictionary (129 fields documented)
- Chronological curation log (14 documented decisions)

## 3. Dataset Description

### Unit of Observation
Each record in `CLEANED_RECORDS.csv` represents a **facility × chemical × year** combination - the original TRI reporting unit. This preserves the regulatory structure of the source data.

### Core Fields (selected)

| Field | Description |
|---|---|
| `TRIFID` | Unique TRI facility identifier (location-based, stable across ownership changes) |
| `YEAR` | Reporting year (2010–2024) |
| `CHEMICAL` | Chemical name (may be generic if trade-secret claimed) |
| `TOTAL_RELEASES` | On-site + off-site release total (pounds; grams for dioxins) |
| `FORM TYPE` | `R` = full Form R; `A` = Form A Certification |
| `INDUSTRY SECTOR` | NAICS-based sector classification |
| `CLASSIFICATION` | `TRI` / `PBT` / `DIOXIN` |
| `CARCINOGEN` | OSHA carcinogen flag |
| `PFAS` | Per- and polyfluoroalkyl substance flag |

### Curation Flags

| Flag | Meaning |
|---|---|
| `FLAG_ZERO_RELEASE` | `TOTAL_RELEASES == 0` |
| `FLAG_FORM_A` | Form A submission |
| `FLAG_MISSING_RELEASE` | `TOTAL_RELEASES` is null (0 records in this dataset) |
| `FLAG_SUDDEN_CHANGE` | Year-over-year change >100% for a facility-chemical pair |
| `FLAG_CARCINOGEN` | OSHA-classified carcinogen |
| `FLAG_PBT` | Persistent, bioaccumulative, and toxic chemical |
| `FLAG_PFAS` | Per- and polyfluoroalkyl substance |

### Derivative Datasets

| File | Description | Grain |
|---|---|---|
| `FACILITY_TIME_SERIES.csv` | Annual totals per facility with quality flag counts | Facility × Year |
| `INDUSTRY_TIME_SERIES.csv` | Annual totals by NAICS industry sector | Sector × Year |
| `CHEMICAL_SUMMARY.csv` | All-years release totals, ranked by total release | Chemical |
| `CHEMICAL_YEAR_TIME_SERIES.csv` | Release trends per chemical per year | Chemical × Year |
| `YEAR_STATS.csv` | Dataset-level statistics per year | Year |

## 4. Curation Decisions

### Zero Value Semantics

**21.3% of records have `TOTAL_RELEASES == 0`.** This is expected and interpretable, not a data error.

Per EPA Basic Data Files Documentation, zeros arise from three sources:
1. **"Not Applicable" (NA)** - facility has no physical pathway for that release (e.g., no water body nearby)
2. **Pre-electronic blanks** - paper form submissions prior to TRI-MEweb (primarily pre-2014) left quantity fields blank; EPA inserted zeros
3. **Form A submissions** - Form A certifies total release < 500 lbs; no quantity detail is required; all fields are zero by design

**Decision:** Zeros are retained and flagged (`FLAG_ZERO_RELEASE`, `FLAG_FORM_A`). They are never imputed or removed. Users who want to exclude them can filter on these flags.

### Empty Fields Retained

Ten fields are 100% empty in the Illinois 2010–2024 dataset:
`BIA`, `TRIBE`, `PRIMARY SIC`, `SIC 2–6`, `HORIZONTAL DATUM`

These are retained because:
- They are part of the standardised TRI schema
- Their emptiness reflects scope (Illinois is non-tribal; SIC replaced by NAICS after 2005)
- Removing them would break schema compatibility with the national TRI data model

### Sudden Change Flag

Records with >100% year-over-year change in `TOTAL_RELEASES` for a given facility-chemical pair are flagged (`FLAG_SUDDEN_CHANGE`). These are **not removed** - sudden changes may reflect genuine events (one-time releases), facility expansions, or reporting corrections. Analysts should review them contextually.

## 5. Known Limitations

| Limitation | Detail |
|---|---|
| **PFAS scope** | PFAS chemicals added to TRI in RY 2020. Pre-2020 records have no PFAS data by design. |
| **POTW methodology break** | POTW release/treatment calculation changed in RY 2014. Pre-2014: metals assumed 100% released. Post-2014: facilities report % breakdowns. Time-series comparisons on POTW fields should account for this. |
| **Self-reported estimates** | TRI releases are primarily facility-estimated, not directly measured. Estimation methods vary. |
| **Threshold reporting** | Only facilities above EPCRA thresholds report. TRI data represents reported releases, not total statewide emissions. |
| **Facility turnover** | Facilities open, close, merge, or fall below thresholds between years. Changes in record count do not necessarily reflect changes in actual emissions. |
| **NAICS consistency** | NAICS codes for pre-2006 data were assigned by EPA (not self-reported). From 2006 onward, facilities self-report. Assignment methods documented in TRI Basic Data Files Appendix D. |
| **Trade secrets** | Some chemical identities are listed as generic names. CAS `9999999999` indicates a sanitised trade-secret submission. |
